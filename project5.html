Challenge:

Every time we started a new project or started a new phase of the project we were creating projects from scratch. This would mean setting up a new components library each time.
This meant we were designing the same components over and over again with the style changing each time to suit functionality. We needed to:

  • Recreate old components
  • Gather all old use cases and create use cases for each component

Beyond Pixels & Styles: Craft Experiences with Design Systems

Our journey began by establishing a well-defined hierarchy for our design system library. This strong foundation, built on essential components like color, typography, grid, spacing, and shadows, empowers us to create highly scalable products. With this robust base, we gain the flexibility to leverage any foundation library in the future when building new design system components for various applications, including provider apps and SaaS products.

Foundation Time with Figma Tokens:

After collaborative discussions and brainstorming, we've refined our design system vision and are now building a robust foundation using Figma's variables and tokens. This ensures consistency and efficiency when creating future components, accessible through our centralized foundation library.
